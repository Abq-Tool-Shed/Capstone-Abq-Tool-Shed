Relationships

Users to Tools: 1 to many
Each tool is for one user only (no splitting in our example)
One user can add multiple tool listings
Borrow to Tools: 1 to many
Each tool can be borrowed by one user at a time
One user can borrow multiple tool at a time
Rating to User: many-to-many
Each User can assign many ratings to many users
We have to create a weak entity to encapsulate this relationship
