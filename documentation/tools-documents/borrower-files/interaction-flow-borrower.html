Interaction Flow

User story: Barbra streisand is looking to borrow a chainsaw to cut fire wood for her grandfather.

1: Barbra Streisand logs onto his/her account using email and password

2: The application takes entered data and logs user into account

3: Barbra Streisand then looks at posts from lender users

4: Barbra Streisand finds a lender user that has a post for a chainsaw

5: Barbra Streisand clicks on lender users post

6: The application loads the lender users post

7: Barbra Streisand then uses in-application email form to borrow the lender users chainsaw

8: The application then process and sends in-application email form data to lender user

9: The Lender user receives in application message and agrees to transaction

10: The lender and borrower meet in real life and transaction is half way completed

11: Barbra Streisand logs back onto the application

12: The application loads users input data and logs user into application

13: Barbra Streisand goes back to the lender users profile and submits in-application email form to return tool

14: The application submits the data and sends in-application message to lender user

15: Lender user gets back tool that was lent out in real life

16: The application then submits data that the transaction was successful
